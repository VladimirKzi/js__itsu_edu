# Практична 11. Класи та об'єктно-орієнтоване програмування
## Варіант 1: Система управління транспортом

## Архітектура класів
- **Vehicle (Base)**: Базовий клас з інкапсульованими даними про пробіг та пальне.
- **Car, Truck, Motorcycle**: Спадкоємці, що реалізують специфічні властивості та поліморфний розрахунок вартості обслуговування.
- **Fleet**: Менеджер-клас для агрегації об'єктів та збору статистики.

## Реалізовані принципи ООП
- **Інкапсуляція**: Використання `#privateFields` для захисту критичних даних (пробіг).
- **Наслідування**: Використання `extends` та `super()` для розширення функціоналу.
- **Поліморфізм**: Перевизначення методу `getMaintenance()` для різних типів транспорту.
- **Абстракція**: Виокремлення спільних рис транспорту в базовий клас.

## Як запустити
- 1. Склонуйте репозиторій.
- 2. Відкрийте `index.html` у вашому браузері.
