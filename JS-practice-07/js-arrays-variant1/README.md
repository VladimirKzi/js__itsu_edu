# Практична 7. Масиви
## Варіант 1: Система управління студентами + Сортування

## Функціональні можливості
- 1. **Керування студентами**: Додавання, валідація даних, фільтрація за факультетом та мінімальним GPA за допомогою методів `filter` та `map`.
- 2. **Пошук студента**: Пошук студента по імені через `find`.
- 3. **Динамічна статистика**: Розрахунок середнього бала через `reduce`, визначення ТОП-3 студентів та автоматичний розподіл кількості осіб по факультетах.
- 4. **Порівняння алгоритмів**: Власна реалізація алгоритмів `Bubble Sort` та `Quick Sort` з нуля з інтегрованими лічильниками порівнянь.

## Результати аналізу алгоритмів
- При тестуванні на масивах різного розміру зафіксовано кардинальну різницю у швидкості та кількості порівнянь:
- **Bubble Sort** демонструє складність O(n^2), через що на 5000 елементів кількість порівнянь сягає ~12.5 мільйонів операцій.
- **Quick Sort** демонструє середню складність O(n \log n), обробляючи 5000 елементів майже миттєво.
