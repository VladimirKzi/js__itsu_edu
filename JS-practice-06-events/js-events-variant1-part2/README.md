# Практична робота №6 (Частина 2): Complex Interactive Apps
## Варіант 1: Kanban Board with Drag & Drop

### Опис проєкту
Цей застосунок є повноцінною Kanban-дошкою для керування завданнями. Проєкт демонструє роботу зі складними подіями JavaScript, зокрема Native Drag and Drop API, обробку масивів даних (CRUD) та синхронізацію зі сховищем браузера.

### Основний функціонал
- 1.  **Drag & Drop система:**
-     * Вільне перетягування карток між колонками (To Do, In Progress, Done).
-     * Зміна порядку карток усередині однієї колонки (Reordering).
-     * Візуальна індикація зони скидання (Drop zone highlight).
- 2.  **Повний цикл CRUD:**
-     * **Create:** Додавання завдань через модальне вікно.
-     * **Read:** Відображення деталей (пріоритет, виконавець, дедлайн, опис).
-     * **Update:** Редагування будь-якої картки за допомогою подвійного кліку.
-     * **Delete:** Видалення окремих карток.
- 3.  **Пошук та Фільтрація:**
-     * Real-time пошук за заголовком та описом (оптимізовано через **Debounce**).
-     * Фільтрація завдань за рівнем пріоритету.
- 4.  **Збереження даних:**
-     * Автоматичне збереження стану дошки в **LocalStorage**. Дані залишаються після оновлення сторінки.
- 5.  **Статистика:**
-     * Автоматичний підрахунок кількості завдань у заголовку кожної колонки.

### Технічні рішення
- **Оптимізація:** Використано `throttle`/`debounce` для обробки подій пошуку та перетягування.
- **Event Delegation:** Обробка видалення та редагування реалізована через делегування подій на контейнерах списків.
- **Color Coding:** Пріоритети (High, Medium, Low) мають відповідне кольорове кодування для швидкого візуального аналізу.
- **UI/UX:** Плавні анімації переміщення карток та інтерактивні ховер-стани.

### 📂 Структура проєкту
- `index.html` — архітектура дошки, колонок та модальних вікон.
- `style.css` — стилізація карток, станів перетягування (`drag-over`) та адаптивна сітка.
- `script.js` — логіка Drag & Drop API, керування масивом `tasks` та робота з LocalStorage.
