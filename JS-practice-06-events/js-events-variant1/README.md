# Практична робота №6: Події та DOM маніпуляції
## Варіант 1: Форма з live-валідацією та real-time пошуком

### Опис проєкту
Цей проєкт демонструє роботу з обробниками подій в JavaScript, маніпуляції деревом DOM та створення інтерактивного користувацького інтерфейсу. Основна увага приділена продуктивності (використання **Debounce** та **Event Delegation**) та доступності (**A11y**).

### Функціональні можливості
- 1.  **Контактна форма:**
-     * **Live-валідація:** перевірка імені, email, телефону та довжини повідомлення в реальному часі.
-     * **Progress Bar:** візуалізація прогресу заповнення форми (0-100%).
-     * **UX Feedback:** динамічна зміна кольорів рамок, поява іконок успіху/помилки та зрозумілих підказок.
-     * **Smart Submit:** кнопка відправки автоматично розблоковується лише після успішної валідації всіх полів.
- 2.  **Компонент пошуку:**
-     * **Real-time фільтрація:** пошук серед 30+ студентів з миттєвим результатом.
-     * **Підсвічування тексту:** знайдені збіги виділяються візуально.
-     * **Обробка порожніх результатів:** відображення повідомлення "Нічого не знайдено".

### Технічні особливості
-  **Event Delegation:** обробка кліків на списку результатів реалізована через одного слухача на батьківському елементі `<ul>`.
-  **Debounce (300ms):** оптимізація навантаження при введенні даних у поля форми та пошуку.
-  **Доступність (A11y):** використання ARIA-атрибутів для прогрес-бару та забезпечення навігації по списку за допомогою клавіатури (`Tab`, `Enter`).
-  **Адаптивність:** верстка на основі Grid та Flexbox, що коректно відображається на мобільних пристроях.
-  **Анімації:** плавні CSS-переходи для станів валідації та прогрес-бару.

### Структура проєкту
-  `index.html` — розмітка сторінки.
-  `style.css` — стилізація, адаптивна сітка та анімації.
-  `script.js` — логіка валідації, пошуку та маніпуляцій з DOM.
